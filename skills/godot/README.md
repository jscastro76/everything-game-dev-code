# godot

Godot-specific implementation skills covering structure, GDScript/C#, scene architecture, testing, performance, signals, tooling, release, and resources.

## Included skills
- `godot-project-structure`: Organize Godot project files so scenes, scripts, resources, addons, tests, and export assets stay predictable.
- `godot-gdscript-standards`: Apply GDScript standards that keep scene-driven projects readable, maintainable, and safe to refactor.
- `godot-csharp-patterns`: Use C# in Godot with clear interop boundaries, node ownership, and engine lifecycle awareness.
- `godot-scene-architecture`: Structure scenes, nodes, autoloads, and resources so Godot projects remain scalable and understandable.
- `godot-testing`: Choose effective automated and manual test layers for Godot features without overcomplicating the project.
- `godot-performance`: Profile Godot runtime, scene, rendering, and resource behavior based on representative builds and content.
- `godot-signals-patterns`: Use signals for decoupling without creating hidden event webs, ownership confusion, or lifecycle bugs.
- `godot-editor-tooling`: Create Godot editor tools, inspectors, import helpers, and validation flows that reduce manual authoring risk.
- `godot-build-release`: Run Godot export, CI, and release packaging with reproducible presets and platform-aware checks.
- `godot-resource-management`: Manage Godot resources, scenes, and imported assets so references, loading, and sharing behavior remain safe and efficient.
