# workflow

Skills for planning, documentation, review loops, and multi-agent coordination.

## Included skills
- `gdd-writing`: Create or update a Game Design Document that is actionable for design, engineering, QA, production, and content teams.
- `technical-design-document`: Turn feature intent into a Technical Design Document that is implementable, reviewable, and testable.
- `vertical-slice-planning`: Define a vertical slice that proves gameplay, quality bars, and production viability without over-building the full game.
- `milestone-planning`: Build milestone plans that are dependency-aware, risk-aware, and tied to real delivery outcomes.
- `risk-register`: Identify, track, and revisit the risks most likely to derail scope, quality, budget, or release readiness.
- `tdd-workflow`: Apply test-driven development or test-first thinking to gameplay, tooling, and high-risk technical work.
- `verification-loop`: Run a structured verification pass that checks behavior, edge cases, quality bars, and documentation alignment.
- `playtest-analysis`: Turn raw playtest observation into actionable design, UX, tuning, and production decisions.
- `continuous-learning`: Capture lessons learned from implementation, production, QA, and release so the project improves over time.
- `orchestration-patterns`: Coordinate multiple agents and disciplines so complex tasks are broken down cleanly and handed off without ambiguity.
