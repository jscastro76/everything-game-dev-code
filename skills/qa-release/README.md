# qa-release

Quality, compliance, submission, crash handling, and release-readiness skills.

## Included skills
- `qa-test-matrix`: Build a coverage matrix that shows what must be tested, on which configurations, and at what milestone confidence level.
- `bug-triage`: Classify bugs by player impact, reproducibility, severity, and milestone risk so the team fixes the right things first.
- `release-readiness`: Evaluate whether a build is genuinely ready for external testing, certification, store submission, or launch.
- `console-certification`: Prepare the project for console platform requirements, TRCs/TCRs/XRs, and first-party submission expectations.
- `crash-triage`: Handle crashes as high-severity defects with reproducibility, symbol quality, clustering, and release impact in mind.
- `store-submission`: Prepare store-facing assets, metadata, compliance paperwork, and packaging details for submission.
- `compliance-checklists`: Create operational checklists for legal, accessibility, privacy, platform, and product compliance obligations.
