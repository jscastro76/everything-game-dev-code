# unreal

Unreal-specific implementation skills covering structure, C++, Blueprints, framework usage, testing, performance, replication, tooling, and release.

## Included skills
- `unreal-project-structure`: Organize Unreal code, plugins, content, and config so ownership and build behavior stay clear.
- `unreal-cpp-standards`: Apply C++ standards that fit Unreal lifecycle, reflection, modules, and gameplay framework concerns.
- `unreal-blueprint-patterns`: Use Blueprints for iteration and authored behavior without letting graph sprawl become hidden architecture.
- `unreal-gameplay-framework`: Use the Unreal gameplay framework intentionally so authority, ownership, and system lifetime are understandable.
- `unreal-testing`: Choose effective Unreal automation and integration test layers without over-relying on brittle map-level tests.
- `unreal-performance`: Profile packaged Unreal builds on target hardware and connect fixes to the real bottleneck domain.
- `unreal-replication`: Implement and review Unreal replication with explicit ownership, relevancy, authority, and failure behavior.
- `unreal-gas-patterns`: Use Gameplay Ability System or equivalent ability frameworks with clear ownership, data flow, and tuning hooks.
- `unreal-editor-tooling`: Create Unreal editor tools, validators, utility widgets, or commandlets that reduce manual work and pipeline risk.
- `unreal-build-release`: Run Unreal builds, packaging, CI, and release packaging with reproducible configuration and platform-aware checks.
