## Steps

### 1. Load the review context

In your new Claude Code session, tell Claude to read and apply the review context and the milestone plan:

```
Read contexts/review.md and apply it to this session. Also read the milestone plan — it is likely at design/MILESTONE-PLAN.md or a similar path in your project — and use it to validate exit criteria as we complete each review step.
```

This activates `unity-reviewer`, `code-reviewer`, `refactor-cleaner`, and `doc-updater`.
The production-phase agents (`gameplay-programmer`, `ui-programmer`) move to a support role:
they can answer questions but do not write new code during the review pass.

Consult `docs/orchestration/role-handoffs.md` to understand how the review agents hand off
to each other in this phase.

### 2. Run the full project review — /unity-review

```
/unity-review Review "Dash & Collect" — a 2D endless runner in Unity. Systems: GameManager, PlayerController, SpawnManager, CollisionHandler, ScoreManager, AudioManager, SaveSystem, InputHandler, UI controllers. Check architecture, event subscriptions, GC allocations, serialization, and prefab hygiene.
```

The `unity-reviewer` agent uses `skills/unity/unity-project-structure` and
`skills/unity/unity-performance`, applying all `rules/unity/` files.

**Review checklist (generated by the agent):**

**Architecture:**
- [ ] GameManager singleton uses static Instance, not FindObjectOfType
- [ ] All inter-system communication goes through events, not direct references
- [ ] No circular dependencies between assemblies
- [ ] Input abstraction via IInputProvider — no direct Input calls in gameplay code
- [ ] AudioManager is DontDestroyOnLoad, not re-instantiated on scene load

**Code quality:**
- [ ] No `Update()` loops that could be replaced with event subscriptions
- [ ] No per-frame GC allocations (no `new`, no LINQ, no string concatenation in hot paths)
- [ ] All event subscriptions have matching unsubscriptions in OnDestroy
- [ ] No `public` fields on MonoBehaviours that should be `[SerializeField] private`

**Unity specifics (`rules/unity/coding-style.md`):**
- [ ] `[SerializeField]` used instead of `public` for inspector-exposed fields
- [ ] `RequireComponent` attributes on MonoBehaviours that depend on other components
- [ ] No hardcoded string tags or layer names (use constants or enums)
- [ ] Coroutines used only where necessary (state machine logic is synchronous)

**Object pooling:**
- [ ] SpawnManager pool sizes are configurable via inspector
- [ ] Pooled objects are properly reset on return (position, velocity, active state)

### 3. Run the scene audit — /unity-scene-audit

```
/unity-scene-audit MainMenu
/unity-scene-audit Gameplay
/unity-scene-audit GameOver
```

The `unity-reviewer` applies `rules/unity/prefabs-scenes.md` to each scene.

**Scene audit checklist:**

```
MainMenu.unity
  [ ] No missing script references
  [ ] No missing prefab references
  [ ] Camera is set to the correct resolution reference
  [ ] Only one EventSystem in the scene
  [ ] Canvas render mode is correct (Screen Space - Overlay)
  [ ] All UI text uses the project's defined font asset

Gameplay.unity
  [ ] Player prefab is an instance, not a modified scene object
  [ ] SpawnManager spawn points are correctly positioned (off screen right)
  [ ] Physics layers are correctly assigned (Player, Obstacle, Coin, Ground)
  [ ] No stray GameObjects with placeholder names (e.g. "GameObject", "Sphere")
  [ ] AudioManager is present (or loaded by GameManager via DontDestroyOnLoad)

GameOver.unity (or overlay object)
  [ ] Overlay is correctly deactivated at scene start
  [ ] Retry and Main Menu buttons have correct listeners wired
```

**Rules applied:**
- `rules/unity/prefabs-scenes.md` — no inline modifications to prefab instances in scenes
- `rules/unity/version-control.md` — scene files must not contain GUID-only references
  without corresponding asset files

### 4. Remove dead code — /refactor-clean

```
/refactor-clean Scan "Dash & Collect" for: unused using directives, empty MonoBehaviour methods, scripts not referenced by any scene or prefab, commented-out code, and Debug.Log calls not wrapped in a debug flag. Propose deletions before removing anything.
```

The `refactor-cleaner` agent scans for:
- Unused `using` directives
- Empty MonoBehaviour methods (`void Start() {}`, `void Update() {}`)
- Scripts not referenced by any scene or prefab
- Commented-out code blocks older than the last commit
- `Debug.Log` calls not wrapped in `#if UNITY_EDITOR` or a debug flag

The agent proposes deletions and waits for confirmation before removing anything.

### 5. Sync documentation — /update-docs

```
/update-docs Sync documentation for "Dash & Collect": compare implemented systems against GDD and TDD, check Audio Bible SFX event names match code, check milestone plan dates. Generate a diff of changes needed — do not write anything until approved.
```

The `doc-updater` agent compares the implemented systems against the GDD and TDD,
applying `rules/common/documentation.md`.

**Common drift to check:**

| Document | What may have drifted |
|----------|----------------------|
| GDD | Core loop description if jump feel was tuned during implementation |
| TDD | System responsibilities if architecture changed during production |
| Audio Bible | SFX event names if they were renamed in code |
| Milestone plan | Actual milestone dates vs planned dates |

The `doc-updater` generates a diff of changes needed, not the changes themselves.
A human (you) approves each documentation update before it is written.

### 6. Review the orchestration docs

Now that a full production + review cycle has been run, read:

- `docs/orchestration/command-agent-map.md` — confirm all commands used in Chapters 3–5
  match the agents that were actually invoked
- `docs/orchestration/role-handoffs.md` — note any handoff points that were unclear or
  that required manual intervention; these are candidates for scaffold improvements

## Common Issues Found in Review

These are the most frequent issues the `unity-reviewer` flags in projects of this type:

| Issue | Rule Violated | Fix |
|-------|--------------|-----|
| `FindObjectOfType` in Awake | `rules/unity/patterns.md` | Use static Instance or inject via inspector |
| Missing OnDestroy unsubscription | `rules/unity/coding-style.md` | Add `OnDestroy()` with event -= handler |
| String.Format in score HUD Update | `rules/common/coding-style.md` | Cache format string or use event-driven update |
| Prefab instance modified in scene | `rules/unity/prefabs-scenes.md` | Revert instance override, fix in prefab |
| Empty Update() left from template | `rules/unity/coding-style.md` | Delete empty lifecycle methods |
